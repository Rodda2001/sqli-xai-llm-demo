## Detailed Changes

### 1. Shared Preprocessing (`preprocessing.py`)
- `normalize_query()`: whitespace collapse, safe URL decode, lowercase
- `extract_structural_features()`: 10 lightweight features (query length, quote counts, comment markers, semicolons, suspicious keywords, functions, operators, hex patterns)
- Same module used in training and inference — zero drift

### 2. Dataset Preparation (`prepare.py`)
- **Label repair**: recovers labels spilled into unnamed columns (310 rows recovered from SQLiV3.csv)
- **Normalized dedup**: deduplicates on normalized query, not raw string (removed 22,549 near-duplicates)
- **Per-source tracking**: `_source` column preserved for reporting
- **Improved subtype labeling**: expanded regex patterns, added `other` fallback instead of defaulting everything to `auth_bypass`
- Better distribution: auth_bypass went from being the catch-all default (would have been 8000+) to 764 (actual tautology patterns only)

### 3. Feature Engineering
- TF-IDF char n-grams (2-5) preserved as primary features
- Added 10 structural features combined via `scipy.sparse.hstack`
- Features reproduced identically in training and inference via `build_features()` / `build_inference_features()`

### 4. Training (`train_one.py`)
- Full metrics: accuracy, precision, recall, F1, ROC-AUC, CV AUC
- Confusion matrix plots auto-generated
- Classification reports saved as text files
- Sample predictions saved
- SHAP background sample (200 representative training rows) saved as `.npz`
- `meta.json` with threshold and model metadata
- `training_report.json` with complete audit trail
- Small subtype classes merged dynamically (min 50 samples)

### 5. SHAP/XAI Improvements
- **Real background sample**: loaded from `shap_background.npz` (200 training rows) instead of dummy single query
- **SQLi only**: XAI runs only for detected SQLi queries
- **Safe queries**: empty `xai_tokens` list, no misleading text
- SQL-pattern-based feature mapping preserved and improved
- Graceful fallback if SHAP fails

### 6. Safe Query Bug Fix (CRITICAL)
- **Before**: safe queries showed "LLM unavailable" even when LLM was fine
- **After**: safe queries return `llm_explanation: null` (no XAI, no LLM attempted)
- Dashboard handles `null` correctly (existing code already checks `d.llm_explanation`)
- Clear distinction: `null` = not needed vs. actual failure text = LLM tried and failed

### 7. LLM Explanation Quality
- Improved prompt: structured 4-line format (Threat/Reason/Key Signals/Recommendation)
- Strong domain-specific fallback for each attack type (auth_bypass, union_based, blind_time, etc.)
- Fallback explanations are SOC-grade, not generic
- Timeout reduced to 30s (configurable) instead of 120s

### 8. API Robustness
- Global exception handler — never exposes stack traces
- Empty/invalid input returns valid safe JSON
- Query length enforcement (10,000 chars max)
- Model load failure handled gracefully (503 status)
- All errors logged internally

### 9. Configuration
- All paths, thresholds, LLM settings centralized in `config.py`
- Threshold loaded dynamically from `meta.json`
- Environment variable overrides for LLM URL, model, timeout, log level
